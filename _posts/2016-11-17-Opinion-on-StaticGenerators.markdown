Static site generators (SSG) are great overall, any web server can run them, no security issues, they are fast and quick to update.
This means that they are a great starting point for any project, and if the project needs a dynamic site they can always base it off the SSG site.
For example, Jekyll is great when there are many people working on the site, they can work with different files which means that they will not overwrite someone else work.
The problem with SSG is that it has a learning curve, however there is a lot of documentation online so I do not think that it is a big problem.
