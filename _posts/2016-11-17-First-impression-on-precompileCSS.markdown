For me, it took a little while to get into Sass. It might have been that I did not start from the beginning and built up, but instead used the templates Sass files.
Still, as soon as I started to get comfortable in the Sass files there were no problems. Comparing it with the regular way of writing CSS, it feels more comfortable and orderly managed.

I used the variable techniques to connect picture and darken some boarders. I also tried out the new syntax when you can step from a parent class down to a class child ".page-content { .wrapper{}}"
Yet, I feel that there is much more that I can try out. However, I want to start closer from scratch and build up my own code, but that will be in the next versions.

The good parts of Sass is that it now really feels like a programming language.
But, now I also get the feeling of that I am missing out on features and that I am still in the old ways of thinking and solving the problems in an inefficient way.
I sometimes get lost in the files, but that is because I decided to build on the basic templates that Jekyll compiles and not create own ones.
However, now when I have read someone else code for a while and added some code I feel more confident to build my own one.
